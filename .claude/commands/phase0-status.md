Phase 0 の進捗ダッシュボードを生成する。

## 手順

1. **現在の週番号計算**: 開始日 2026-05-08 から経過週(`date` 計算)
2. **Track 別進捗**: 各 `tracks/<X>-*/` 配下の .md ファイル数(README.md と .gitkeep を除外)
3. **decision-doc.md の確定項目**: `synthesis/decision-doc.md` を Read して各 § の "確定" / "暫定" / "未決定" をカウント
4. **直近活動**: 過去 7 日の git log
5. **リスク評価**:
   - 未着手 Track があるか
   - decision-doc の §1, §2, §3, §4, §5 で残期間内に確定可能か

## 出力

```markdown
# Phase 0 Status — YYYY-MM-DD

## 進捗
- Phase 0 Week: W<N> / W12
- 残期間: <X> 週
- 開始日: 2026-05-08
- 目標確定日: 2026-08-初

## Track 別進捗
| Track | 完成度(目安) | ファイル数 | 直近更新 |
|-------|---------------|-----------|---------|
| A     | XX%           | N         | YYYY-MM-DD |
| B     | XX%           | N         | YYYY-MM-DD |
| C     | XX%           | N         | YYYY-MM-DD |
| D     | XX%           | N         | YYYY-MM-DD |

## decision-doc.md 確定状況
- §1 法人設立国: 未決定 / 暫定 / 確定
- §2 ビジネスモデル: ...
- §3 技術アーキテクチャ: ...
- §4 最初の3顧客: ...
- §5 PayPay との関係(Path): ...

## 直近 7 日の活動
- コミット数: X
- 新規ファイル: <list>

## リスク
- ⚠️ <未着手 Track があれば警告>
- ⚠️ <残期間で確定困難な項目があれば警告>

## 次の優先アクション(3 つ)
1. ...
2. ...
3. ...
```

**守ること**:
- 1 画面(50 行以内)に収める
- 「完成度」は目安なので %判定根拠を簡潔に書く
- リスクなしなら "なし" と明記
- 残期間が短いのに未確定項目が多い場合は太字で警告
