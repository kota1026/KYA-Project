# Mastercard × Google — "Verifiable Intent" オープン標準発表

## 要約
Mastercard と Google が共同で「Verifiable Intent」を発表（2026-03-05）。エージェント取引において消費者 ID・指示内容・実行結果を 1 つの改ざん耐性レコードに紐づけ、Selective Disclosure で各当事者に必要最小限の情報だけを開示する。仕様は GitHub でオープンソース公開、Fiserv・IBM・Checkout.com 等がコミットメント表明。Google AP2 / UCP と整合する設計で、プロトコル非依存。

## なぜ KYA に関係あるか
Selective Disclosure + 暗号的監査証跡の組み合わせは、KYA が目指す「エージェント認証 + 委任証明」のネットワーク層実装そのもの。Track A: Mastercard のポジショニング強化、Track C: Selective Disclosure 技術の実用事例として両方で参照すべき。Track A の既存 Mastercard ファイル（mastercard-agent-suite-kya.md）と統合・更新要。
